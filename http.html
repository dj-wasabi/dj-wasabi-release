4. djWasabi.http.request()¶

Basic functions for doing http requests via the requests module.

class djWasabi.http.request(debug: bool = False, status: Optional[list] = None, methods: Optional[list] = None, backoff: int = 1, retries: int = 5, timeout: int = 10, verify: bool = True)[source]¶

__init__(debug: bool = False, status: Optional[list] = None, methods: Optional[list] = None, backoff: int = 1, retries: int = 5, timeout: int = 10, verify: bool = True)[source]¶

Set defaults and setting up retry mechanism for http requests.

Parameters

  • debug (bool) – If we need debug information or not.

  • status (list) – A list with http status code that needs to be retried.

  • methods (list) – A list with htto methods that are allowed to be retried.

  • retries (int) – The amount of retries we want to use.

  • backoff (int) – The backoff timeout. See https://findwork.dev/blog/advanced-usage-python-requests-timeouts-retries-hooks/

  • timeout (int) – The timeout in seconds

  • verify (bool) – If TLS connections needs to verify the remote certificate.
